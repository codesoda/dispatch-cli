## Codebase Patterns
- Config in `src/config.rs`: `resolve_config(cli_cell_id, start_dir)` with inner testable function for env var isolation
- Config file is `dispatch.config.toml`, searched upward from cwd; cell_id derived via `DefaultHasher` on canonical path
- Precedence: CLI `--cell-id` > `DISPATCH_CELL_ID` env > config file > derived fallback
- Use `resolve_config_inner()` in tests to avoid env var races in parallel test execution
- Edition 2021, async everywhere with tokio, lib/bin split
- Use `cargo fmt --check && cargo clippy --all-targets -- -D warnings && cargo build` before committing
- Error types in `src/errors.rs` with thiserror, CLI in `src/cli.rs` with clap derive
- Tracing setup in `src/logging.rs` ‚Äî rolling file appender to data_local_dir + stderr layer
- All dependencies: clap, serde, toml, thiserror, tracing, tracing-subscriber, tracing-appender, tokio, reqwest, semver, uuid, serde_json, dirs
- Protocol types in `src/protocol.rs`: `BrokerRequest` (tagged enum) and `BrokerResponse` (tagged enum with `ResponsePayload`)
- Client in `src/client.rs`: `Client::new(project_root, cell_id)` then `client.send_request(&req).await`
- Broker routes requests in `handle_request()` ‚Äî add new handler logic there per request type
- CLI subcommands (except `serve`) map to `BrokerRequest`, send via `Client`, print JSON response to stdout
- `Worker` struct in `protocol.rs` with id, name, role, description, capabilities, expires_at
- `BrokerState.workers` is `HashMap<String, Worker>` ‚Äî `register_worker()` returns UUID, `evict_expired()` cleans up
- Default worker TTL is 300s (5 min) ‚Äî `DEFAULT_WORKER_TTL_SECS` const in `broker.rs`
- `ResponsePayload` variants added: `WorkerRegistered`, `WorkerList`, `MessageAck`, `HeartbeatAck`, `Message`, `Timeout`, `Data`
- `BrokerState.list_workers()` evicts expired then returns cloned Vec; `heartbeat_worker()` evicts then renews TTL
- `now_secs()` helper in broker.rs ‚Äî use instead of inline SystemTime boilerplate
- Test helpers `send_json_request()` and `register_worker()` in broker tests ‚Äî use for any broker integration test
- `BrokerState.mailboxes` is `HashMap<String, VecDeque<Message>>` ‚Äî `send_message()` queues messages per worker
- `Message` struct in `protocol.rs`: message_id, from (optional), to, body
- `send_message()` evicts expired workers first, returns None if recipient not found
