## Codebase Patterns
- Config in `src/config.rs`: `resolve_config(cli_cell_id, start_dir)` with inner testable function for env var isolation
- Config file is `dispatch.config.toml`, searched upward from cwd; cell_id derived via `DefaultHasher` on canonical path
- Precedence: CLI `--cell-id` > `DISPATCH_CELL_ID` env > config file > derived fallback
- Use `resolve_config_inner()` in tests to avoid env var races in parallel test execution
- Edition 2021, async everywhere with tokio, lib/bin split
- Use `cargo fmt --check && cargo clippy --all-targets -- -D warnings && cargo build` before committing
- Error types in `src/errors.rs` with thiserror, CLI in `src/cli.rs` with clap derive
- Tracing setup in `src/logging.rs` — rolling file appender to data_local_dir + stderr layer
- All dependencies: clap, serde, toml, thiserror, tracing, tracing-subscriber, tracing-appender, tokio, reqwest, semver, uuid, serde_json, dirs
- Protocol types in `src/protocol.rs`: `BrokerRequest` (tagged enum) and `BrokerResponse` (tagged enum with `ResponsePayload`)
- Client in `src/client.rs`: `Client::new(project_root, cell_id)` then `client.send_request(&req).await`
- Broker routes requests in `handle_request()` — add new handler logic there per request type
- CLI subcommands (except `serve`) map to `BrokerRequest`, send via `Client`, print JSON response to stdout
- `Worker` struct in `protocol.rs` with id, name, role, description, capabilities, expires_at
- `BrokerState.workers` is `HashMap<String, Worker>` — `register_worker()` returns UUID, `evict_expired()` cleans up
- Default worker TTL is 300s (5 min) — `DEFAULT_WORKER_TTL_SECS` const in `broker.rs`
- `ResponsePayload` variants added: `WorkerRegistered`, `WorkerList`, `MessageAck`, `HeartbeatAck`, `Message`, `Timeout`, `Data`
- `BrokerState.list_workers()` evicts expired then returns cloned Vec; `heartbeat_worker()` evicts then renews TTL
- `now_secs()` helper in broker.rs — use instead of inline SystemTime boilerplate
- Test helpers `send_json_request()` and `register_worker()` in broker tests — use for any broker integration test
- `BrokerState.mailboxes` is `HashMap<String, VecDeque<Message>>` — `send_message()` queues messages per worker
- `Message` struct in `protocol.rs`: message_id, from (optional), to, body
- `send_message()` evicts expired workers first, returns None if recipient not found
- `BrokerState.notifiers` is `HashMap<String, Arc<Notify>>` — `get_notifier()` returns per-worker Notify handle
- Long-poll pattern: lock state, check mailbox, get notifier, unlock, then `tokio::time::timeout(notifier.notified())` to wait
- `send_message()` calls `notify.notify_one()` after queuing — wakes any long-polling `listen` handler
- `Listen` handler renews worker TTL implicitly — no separate heartbeat needed during listen loops
- `pop_message()` pops front of VecDeque for FIFO ordering
- Example scripts go in `examples/` — shell scripts should be executable, have `--help`, and use `dispatch send` (not internal APIs)

## 2026-04-10 - US-008
- What was implemented: Listen loop with long-poll — `BrokerState.notifiers` HashMap with per-worker `Arc<Notify>` for wakeup signaling, `pop_message()` method for FIFO mailbox consumption, `get_notifier()` for per-worker Notify handle creation, `send_message()` now calls `notify_one()` after queuing, Listen handler with immediate delivery (if message already queued), long-poll wait via `tokio::time::timeout` + `Notify::notified()`, implicit TTL renewal on listen, distinct JSON shapes for message vs timeout responses
- Files changed: src/broker.rs
- **Learnings for future iterations:**
  - `tokio::sync::Notify` is the right primitive for long-poll wakeup — `notify_one()` stores a permit so the waiter wakes even if notification happens between lock release and `notified().await`
  - Must release the Mutex lock before awaiting the Notify — otherwise `send_message` can't acquire the lock to queue the message and notify
  - Pattern: lock → check mailbox → get notifier → drop lock → await notifier with timeout → re-lock → pop message
  - `Notify` doesn't have `try_recv` — use `Arc::ptr_eq` to test notifier identity instead
  - Timeout of 0 falls back to DEFAULT_LISTEN_TIMEOUT_SECS (30s) to avoid instant return
  - 10 new tests: 4 unit (pop FIFO, empty mailbox, unknown worker, notifier identity) + 6 integration (immediate delivery, timeout, long-poll delivery, TTL renewal, unknown worker error, FIFO ordering)
